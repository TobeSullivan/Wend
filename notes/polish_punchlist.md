1. ✅ **White corner artifacts (home / main menu).** INVESTIGATED — NOT REPRODUCIBLE IN RENDER (2026-06-08, CC). Captured the home screen at 1080p + 8× crops of all four corners (`tools/home_shot.{gd,tscn}`): every screen corner is clean dark grass, zero white pixels. Combined with the in-game board corners (also clean), the in-engine render has no white anywhere. The reported "few px of white that reads like rounded corners" is consistent with **Windows 11's OS-level window-corner rounding** in windowed mode (the rounded window mask anti-aliases against the light desktop behind it) — a compositor effect, not the game's render, and absent in fullscreen. Cosmetic, OS-level, windowed-dev-only. No code change; re-confirm in fullscreen if it ever recurs.

2. ✅ **Text too thin.** ACTUALLY FIXED 2026-06-08 (CC) — earlier "done" was wrong. Root cause found via a side-by-side weight capture (`tools/font_sample.{gd,tscn}`): `fredoka_bold.tres` set `variation_opentype = { "wght": 600 }` with a **STRING key**, which Godot **silently ignores** — the variable font kept rendering at its Light (300) default, so every screen looked thin. Fix: the key must be the **integer OpenType tag** — `{ 2003265652: 700.0 }` (`0x77676874` = `'wght'`), bumped to **700/Bold** for a decisively thicker look. `app_theme.tres` already distributes it + the dark outline (`#1a2012`, size 3) app-wide. Verified in the home capture: WEND + all labels now render Bold. (Memory `reference_godot_variable_font_weight` corrected — it had wrongly claimed the string key resolves.)
