# ASSET CATALOG - GDS store, mapped to Wend slots

Durable slot-mapped index of the Game Developer Studio catalogue (1277 packs,
one artist, one coherent style). Built once from a full scrape; this is the
reference you check when picking a cosmetic, so it is never re-triaged.

- **Raw backstop:** `notes/gds_catalog.csv` (all 1277 packs: name, id, slug, both prices, url).
- **This file:** the theme-relevant packs only, grouped by Wend slot and sub-theme.

## How to read
Each line: `pack name ($standard price [tags])`.
- `FREE+`  = free with the GDS+ membership (GDS+ price is $0.00) -> $0 to pull in.
- `OWNED`  = currently owned. **Inferred from a screenshot (stopgap).** For the
  authoritative owned set, run `python gds_catalog.py library.html` and regenerate.
- `$?`     = older single-asset pack whose price markup the scraper skipped; check
  the pack page if it matters (it usually won't - these are cheap one-offs).

## Two tiers (important)
1. **Index (this file):** pack-level. What exists, what it costs, do we own it.
2. **Contents (per pack):** the individual sprites inside a pack are enumerated
   only when we ADOPT that pack, by reading the downloaded files - the store does
   not expose a per-file manifest, so there is no store-wide contents catalogue.

## Selection rules (from `design/COSMETICS.md` + the level-map finding)
- Cosmetics are visual-only, never enter the sim, FX match the default silhouette+duration.
- **Prefer in-pack primitives; standardize on one source.** The level-map pack
  already ships the **1/2/3-star groups, 5-colour flag markers, and 5-colour
  buttons** - move the UI star-rating art, player/team markers, and buttons onto
  these instead of mixing bespoke sources. Coherence is automatic within a pack.
- **Dedupe:** many packs are the same thing re-themed. Pick one canonical per need;
  do not treat every recolour as a distinct catalogue entry.

## Defaults currently in-repo (the skin slot's "stock")
tower = arrow box  ·  board = summer grass  ·  mob = undead  ·  projectile = arrow
